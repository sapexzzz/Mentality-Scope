# Mentality Scope — Build Guide v1.5.0

## 🚀 Quick Start

### Requirements
- **Java:** OpenJDK 17+ (or JBR from Android Studio)
- **Android SDK:** 31 (Min) and 35 (Target)
- **Gradle:** 8.x (wrapper included in project)

### Build from Command Line

```bash
cd /home/mentality/scripts/android_application

# Debug APK
JAVA_HOME=/opt/android-studio/jbr ./gradlew assembleDebug

# Release APK (requires keystore signing)
JAVA_HOME=/opt/android-studio/jbr ./gradlew assembleRelease

# Run lint check (release)
JAVA_HOME=/opt/android-studio/jbr ./gradlew lintRelease
```

> **Important:** `JAVA_HOME=/opt/android-studio/jbr` is required, otherwise Gradle
> may use an incompatible JVM from the system.

### Output Files

```
app/build/outputs/apk/debug/app-debug.apk
app/build/outputs/apk/release/app-release.apk
```

### Install on Device

```bash
adb install app/build/outputs/apk/debug/app-debug.apk
```

## ⚠️ Lint Configuration

In `app/build.gradle.kts` one lint rule is disabled:

```kotlin
lint {
    disable += "MutableCollectionMutableState"
    abortOnError = true
}
```

Reason: known NPE bug in `MutableCollectionMutableStateDetector` from AGP.
This does not affect code quality — the rule is not applicable to this project.

## 🔴 Common Issues

### `Supplied javaHome is not a valid folder`
```bash
# Fix: explicitly set JAVA_HOME
JAVA_HOME=/opt/android-studio/jbr ./gradlew assembleDebug
```

### `Lint found fatal errors while assembling a release target`
- Ensure `app/build.gradle.kts` contains a `lint { ... }` block
- Run: `JAVA_HOME=/opt/android-studio/jbr ./gradlew assembleRelease`

### `MissingDefaultResource` in values-night/colors.xml
- `values/colors.xml` must contain fallback declarations for all colors from `values-night`
- The current project already includes them (`dark_primary`, `dark_primary_dark`, `dark_secondary`, `dark_tertiary`)

### Clear Cache
```bash
JAVA_HOME=/opt/android-studio/jbr ./gradlew clean
JAVA_HOME=/opt/android-studio/jbr ./gradlew assembleDebug
```
